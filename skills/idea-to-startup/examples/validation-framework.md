# Framework de Validación Rápida — 5 Preguntas Binarias
# Valida o descarta una idea de startup en < 1 hora

## Cómo usar este framework

Responde cada pregunta con **Sí** o **No** basado en evidencia, no en intuición.
Si 3 o más respuestas son **No** → STOP. Rediseña la idea antes de continuar.

---

## Las 5 Preguntas

### 1. ¿Resuelve un problema real por el que la gente *paga* actualmente?
**Criterio de éxito**: Puedes citar al menos 3 personas que hoy gastan dinero (o tiempo equivalente) en resolver este problema con una solución imperfecta.
- Sí → continúa
- No → ¿Es un problema de "nice to have" o "must have"? Si es nice-to-have, detente.

### 2. ¿Existe un segmento de clientes claramente definible?
**Criterio de éxito**: Puedes describir al cliente ideal en ≤ 2 líneas con atributos mensurables (ej: "Startups B2B SaaS de 10-50 empleados con ARR < $1M").
- Sí → continúa
- No → La idea es demasiado genérica. Nicha más.

### 3. ¿Puedes validar con una landing + waitlist en < 1 semana?
**Criterio de éxito**: El valor diferencial es suficientemente claro para que alguien deje su email sin ver el producto.
- Sí → continúa
- No → El value proposition es débil o complejo. Simplifica el pitch.

### 4. ¿Es materialmente diferente de las 3 soluciones más cercanas?
**Criterio de éxito**: Puedes completar: "A diferencia de [competitor], nosotros somos la única solución que [diferenciador único]."
- Sí → continúa
- No → ¿Cuál es el 10x improvement sobre la alternativa existente? Si no lo hay, detente.

### 5. ¿El equipo puede construir el MVP en < 60 días?
**Criterio de éxito**: El MVP puede implementarse con las habilidades actuales del equipo sin funding externo.
- Sí → continúa al Plan de Acción
- No → Identifica el cuello de botella (skill gap, complejidad técnica) y resuélvelo primero.

---

## Resultado y Plan de Acción

| Score | Acción |
|-------|--------|
| 5/5 Sí | Construir MVP ahora. Lanzar en 60 días. |
| 4/5 Sí | Identificar el No y mitigarlo antes de construir. |
| 3/5 Sí | Refinar la propuesta. Regresa cuando tengas 4+ Sí. |
| ≤ 2/5 Sí | Rediseña completamente. Esta idea no está lista. |

## Paths de Validación: B2B vs B2C vs Vertical AI

### B2B SaaS
1. Identificar 10 empresas target → entrevistas en 1 semana
2. Landing page + "Request Demo" → medir click-through rate
3. Piloto pagado (aunque sea beta pricing) con 3 empresas antes de construir
4. Métricas: CAC < 3x MRR, Tiempo de ciclo de ventas < 30 días para SMB

### B2C Consumer
1. Landing page + waitlist → 100 emails = señal de interés real
2. Comunidad pre-lanzamiento (Discord/Reddit) → engagement > comportamiento de compra futuro
3. Freemium + paywall para feature clave → mide conversión de free → paid
4. Métricas: D7 retention > 25%, D30 retention > 10%, payback < 3 meses

### Vertical AI (alta oportunidad 2025-2026)
| Vertical | Señal de mercado | Barrera de entrada |
|----------|-----------------|-------------------|
| **Healthcare** | Documentación clínica, diagnóstico asistido | Regulación HIPAA/GDPR médico |
| **Legal** | Research, contratos, compliance | Conocimiento especializado |
| **Fintech** | Análisis de riesgo, fraud detection | Regulación financiera |
| **Manufacturing** | Predictive maintenance, quality control | Acceso a datos OT/IoT |
| **Education** | Tutoring personalizado, evaluación | Adopción institucional lenta |

**Recomendación**: Comenzar en un vertical donde ya tienes acceso (network, experiencia previa o cliente âncora).
