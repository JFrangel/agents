# Proceso para extender el Design System

1. Define nuevos tokens en `design-tokens.json` y actualiza la documentación.
2. Crea el componente en `components/` siguiendo el patrón de Button.
3. Añade ejemplos de uso en `examples/`.
4. Ejecuta los scripts de validación y accesibilidad.
5. Actualiza el checklist en `check-design.md`.
6. Realiza revisión de código y documentación antes de mergear.
